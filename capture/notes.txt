Topic 4.7 — QoS. Cold question:
What problem does QoS solve and give me one real example where without QoS a network becomes unusable?
Go.I only know QoS means quality of service, and a network becomes unstable when it is not reviewed audited and maintained regularly?That is network maintenance — different concept. Built from scratch.
The problem QoS solves:
A network link has finite bandwidth. When multiple types of traffic compete for that bandwidth simultaneously — voice calls, video, file downloads, email — they all get treated equally. Equal treatment sounds fair but it is actually a problem.
Why equal treatment breaks things:
Link capacity: 100Mbps

Someone starts downloading a 10GB file    → uses 90Mbps
VoIP call also running                    → gets remaining 10Mbps
VoIP needs consistent low latency        → instead gets delayed, jittery
Call becomes choppy, unusable
The file download does not care if its packets arrive 200ms late. The voice call absolutely does — human ears detect delays above 150ms immediately.
QoS solves this by prioritising traffic:
Voice/Video  → highest priority, low latency guaranteed
Critical apps → medium priority
File downloads → lowest priority, gets leftover bandwidth
The file download still completes — just slower. The voice call sounds perfect.
The four QoS mechanisms the exam tests:
Classification — identify and mark traffic type
Marking — tag packets (DSCP values) so every device knows priority
Queuing — separate queues per priority level
Policing/Shaping — control bandwidth per traffic type

Real exam scenario:
A hospital runs VoIP phones and medical imaging file transfers on the same WAN link. Without QoS — a large MRI scan upload saturates the link and doctor phone calls drop. With QoS — VoIP gets guaranteed bandwidth, imaging transfers use whatever remains.
